    关于在新电脑上配置博客这件事。

    唔五一当天的时候收到了新电脑。

    可谓是精挑细选了，当然我一直自信自己的眼光非常的好。

    所以精挑细选后的新电脑可以说是很喜欢了。

    最令我开心的是大屏又很轻，而且近期使用认为续航也到令人开心。

    然后涉及到了换电脑写博客的问题23333

    比起来寒假的时候摸索着弄了半天，新配置的时候非常迅速。

    仍然是基于GitHub pages + Hexo

    因为上一次没有做什么记录这次就写个简单记录吧。

    从零开始

    • Notepad++ [Note ++下载官网（略难开）]（https://notepad-plus.en.softonic.com/）

    [Notepad++下载]（http://dl.pconline.com.cn/html_2/1/117/id=10699&pn=0.html）

    • git [git下载]（https://gitforwindows.org/）

    • Node.js [node.js下载]（https://nodejs.org/en/）

    • hexo Git Bash + npm命令

    任意位置，右键Git Bash 键入：npm install -g hexo

    (hexo -v 可以检测是否安装成功)

    • 初始化Hexo 新建文件夹 + Git Bash

    新创建的文件夹，右键Git Bash 键入：hexo init 安装

    进入_config.yml进行配置:主要是#Site部分吧

    • 部署到Github

    账号,new repository,uesrname/uesrname.github.io

    进入_config.yml进行配置:deploy 部分

    deploy:

    type: git

    repo: https://github.com/uesrname/uesrname.github.io

    branch：master

    (冒号后面一定要加空格)

    • Git配置 Git Bash

    git config –global user.name “你的用户名”
    git config –global user.email “你github的邮箱”

    • 密钥 Git Bash

    ssh-keygen -t rsa -C “你github的邮箱”

    在C：/Users/用户名/.ssh/: 是默认密钥位置

    github设置界面，选择SSH and GPG key，创建一个New SSH key

    打开生成的密钥文件，复制id_rsa.pub里内容，复制到Key，Add SSh key(title随便写)

    • 安装自动部署发布工具 Git Bash + npm命令

    新创建的文件夹，右键Git Bash 键入：npm install hexo-deployer-git –save

    • 发布 Git Bash

    新创建的文件夹，右键Git Bash 依次键入：hexo c,hexo g,hexo d

    登陆github，显示INFO Deploy done： git 完成

    https://username.github.io/ 即可访问

    本次经历

    在deploy部分写地址的时候，https后面多加了空格。

    于是fatal :I don’t handle protocol ‘https’

    注意细节呀。
